# PHP-блог с категориями — Чистый PHP + Smarty + MySQL

https://drive.google.com/file/d/1C9ZfbIElJf_24z_VkiIX_Y6iUsjM6N2c/view?usp=sharing

## Цель

Создать полностью рабочий веб-сайт блога на чистом PHP 8.1+ (без фреймворков) с MySQL, шаблонизатором Smarty, стилями на SCSS и Docker-окружением. Блог поддерживает категории и статьи с подсчётом просмотров, сортировкой, пагинацией и сидером для демо-данных. Существующая Docker-конфигурация, должна быть на чистом PHP.

## Описание

В репозитории находится Docker-конфигурация — `docker-compose.yml`, `Dockerfile`, конфигурация Nginx, Makefile и `.env` файлы. Целевая архитектура (MVC-подобная, Smarty, PDO, SCSS).

### Требования:
- Без фреймворков — чистый PHP с ручной маршрутизацией.
- Smarty 5.x для шаблонов.
- MySQL 8.0 через PDO только с подготовленными запросами.
- SCSS компилируется в CSS (Node.js нужен в Docker-образе или как шаг сборки).
- Docker-окружение для разработки (PHP-FPM + Nginx + MySQL).
- Приветствуется использование возможностей PHP 8.1+ (типизированные свойства, readonly, перечисления, match, именованные аргументы).

### Модель данных:
- `categories` (id, name, description, created_at, updated_at)
- `articles` (id, image, title, description, body, views, created_at, updated_at)
- `article_categories` — связующая таблица (article_id, category_id) для связи многие-ко-многим

### Страницы:
1. **Главная** — каждая категория, в которой есть статьи, 3 последних поста на категорию, кнопка «Все статьи».
2. **Категория** — название, описание, список статей с сортировкой (по просмотрам / по дате) и пагинацией.
3. **Статья** — полная информация о статье, увеличение счётчика просмотров, блок из 3 похожих статей.



Задание:
Требуется реализовать вырожденный интернет-магазин услуг, состоящий из двух страниц - формы авторизации и формы заказа услуги оценки. На выбор пользователю предлагается несколько услуг (например "оценка стоимости автомобиля", "оценка стоимости квартиры", "оценка стоимости бизнеса"), также у пользователя запрашивается электронная почта. При смене услуги пользователю отображается её стоимость (у каждой услуги своя стоимость). При подтверждении заказа он сохраняется в хранилище. Страница заказа доступна только авторизованному пользователю.

Функционал должен быть покрыт тестами, как минимум следующие сценарии:
• при открытии формы заказа не авторизованным пользователем отображается страница с ошибкой
• авторизованному пользователю отображается форма, проверить, что вывелись все
поля и кнопка
• при подтверждении заказа с незаполненной эбектронной почтой или не выбранной услугой пользователю отображается либо страница с ошибкой, либо та же самая форма заказа с текстом ошибки в произвольном месте
• при отправке авторизованным пользователем формой с заполненными полями в хранилище должен появиться новый заказ с данными, соответствующими форме
Требования к реализации:
• авторизацию реализовать средствами symfony security-bundle
• в качестве хранилища использовать файл в произвольном формате, либо БД MySQL
• форма реализуется с помощью шаблонизатора twig + js для изменения стоимости заказа при смене услуги
• внешний вид формы не важен, навыки вёрстки не проверяются, главное - функционал
в качестве тестов использовать тесты приложения с помощью класса WebTestCase, работоспособность js-кода можно не тестировать
